# Правила внесения изменений

1. Создайте Issue с описанием причины изменения.
2. Создайте ветку `model/<issue>-<short-description>` или `docs/<issue>-<short-description>`.
3. Изменяйте inputs только на листе `01_ВВОД`.
4. Проверьте `08_ПРОВЕРКИ`.
5. Обновите документацию, `CHANGELOG.md`, manifest и SHA-256.
6. Откройте Pull Request.
7. После проверки объедините PR в `main`.
8. Для утверждённой версии создайте Tag и Release.

Изменения методики без ADR не принимаются.
