# Управление проектом

## Жизненный цикл изменения

Каждое существенное изменение проходит последовательность:

1. Issue;
2. отдельная ветка;
3. изменение модели или документации;
4. проверки;
5. Pull Request;
6. merge в `main`;
7. закрытие Issue;
8. Tag и Release для утверждённой версии.

## Именование

Примеры веток:

- `scenario/9-revised-dinner-pricing-and-opex`;
- `docs/22-jupyter-book-github-pages`.

Пример тега сценария:

- `s03-v0.1.4`.

## Обновление книги

После первоначальной публикации достаточно изменить файлы в папке `book/`, создать Pull Request и объединить его с `main`. GitHub Actions автоматически пересоберёт и опубликует сайт.

## Контроль публикации

Перед строгой сборкой и развертыванием сайта workflow запускает `scripts/validate_book_content.py`.

Проверка блокирует публикацию, если:

1. вложенный заголовок повторяет нормализованный текст любого родительского заголовка;
2. внешняя HTTP- или HTTPS-ссылка недоступна после повторных запросов;
3. новый раздел отсутствует в собранном сайте.

Правило заголовков применяется ко всем Markdown-страницам книги. Сравнение не учитывает регистр, служебную нумерацию и Markdown-оформление.

Проверка ссылки подтверждает техническую доступность адреса и переходы по перенаправлениям. Для нормативных правовых актов дополнительно требуется содержательная проверка действующей редакции, даты вступления в силу и срока действия документа.

После объединения изменения с `main` workflow фиксирует на опубликованном коммите статус `site/publication`. Во время развертывания статус имеет значение `pending`, после успешного открытия и проверки live-страниц — `success`, а при ошибке публикации или проверки — `failure`. Успешный статус содержит ссылку на опубликованный сайт; ошибочный — на соответствующий запуск GitHub Actions.

## Релизы

Release фиксирует утверждённую версию и должен содержать понятное описание изменений, Excel-модель и SHA-256. Опубликованный файл с тем же номером версии не перезаписывается: исправления выпускаются новой версией.

## Публичность

GitHub Pages является публичным сайтом для публичного репозитория. Перед добавлением материалов следует убедиться, что они не содержат персональных данных, паролей, ключей, банковских реквизитов или конфиденциальных договорных условий.
