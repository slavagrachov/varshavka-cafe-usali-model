# Итоговая конфигурация публикации видео VARSHAVKA

Дата фиксации: **20 июля 2026 года**.

Документ описывает только принятые результаты настройки публикации видеопрезентации проекта VARSHAVKA. Промежуточные эксперименты, временные ссылки и рабочая переписка в состав итоговой документации не включены.

## 1. Назначение решения

Решение обеспечивает публичный просмотр видеопрезентации проекта VARSHAVKA из Telegram и обычного браузера без регистрации и без входа в HeyGen, GitHub или Supabase.

Целевая цепочка публикации:

```text
HeyGen → прямой MP4 → Supabase Storage → GitHub Pages → Telegram
```

- конечный пользователь получает только публичную ссылку на объект Storage.

## 7. GitHub Actions и GitHub Pages

Основной workflow:

```text
.github/workflows/deploy-jupyter-book.yml
```

### 7.1. Используемые actions

```yaml
actions/checkout@v6
actions/setup-node@v6
actions/setup-python@v6
actions/configure-pages@v5
actions/upload-pages-artifact@v4
actions/deploy-pages@v4
```

### 7.2. Автоматические проверки до публикации

Workflow проверяет:

1. установленные версии Python, pip, Node.js и Jupyter Book;
2. доступность публичного MP4 в Supabase;
3. заголовок `Content-Type: video/mp4`;
4. получение непустого диапазона байтов видео;
5. строгую сборку Jupyter Book;
6. наличие файлов:
   - `book/_build/html/index.html`;
   - `book/_build/html/video/index.html`;
   - `book/_build/html/project/index.html`;
7. присутствие постоянной Supabase-ссылки в собранной странице;
8. отсутствие `app.heygen.com` в собранной странице.

### 7.3. Автоматические проверки после публикации

После развертывания workflow:

- ожидает распространения GitHub Pages;
- проверяет главную страницу;
- проверяет `/video/`;
- проверяет `/project/`;
- проверяет наличие Supabase URL на опубликованной странице;
- отклоняет публикацию, если обнаружена ссылка на HeyGen.

## 9. Эксплуатационный регламент

### 9.1. Замена видео без изменения внешних адресов

Для сохранения действующих сообщений Telegram новое видео загружается в тот же объект:

```text
varshavka-video/varshavka-2026-2027.mp4
```

После замены необходимо запустить workflow вручную или внести документированное изменение в `book/**`.

### 9.2. Проверка после замены

Обязательные проверки:

1. прямой MP4 открывается без авторизации;
2. MIME-тип равен `video/mp4`;
3. страница `/video/` содержит постоянную Supabase-ссылку;
4. страница `/project/` открывается;
5. GitHub Actions завершается статусом Success;
6. Telegram открывает ссылку без перехода в HeyGen.

### 9.3. Изменения, требующие обновления документа

Документ обновляется при изменении:

- Supabase project ref;
- имени бакета или объекта;
- публичных URL;
- версии Jupyter Book;
- структуры workflow;
- текста итоговой карточки Telegram;
- длительности или идентификатора видео.
